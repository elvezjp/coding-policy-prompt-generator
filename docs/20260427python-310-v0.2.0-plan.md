# Plan: Python 3.10 へ移行し v0.2.0 をリリース

関連 Issue: [#13](https://github.com/elvezjp/coding-policy-prompt-generator/issues/13)
ブランチ: `tominaga/20260427-python-310-v0.2.0`

## 1. 背景

Dependabot で以下のアラートが発生している。

- [#2 pytest < 9.0.3](https://github.com/elvezjp/coding-policy-prompt-generator/security/dependabot/2)（CVE-2025-71176, Medium）
- [#1 Pygments < 2.20.0](https://github.com/elvezjp/coding-policy-prompt-generator/security/dependabot/1)（CVE-2026-4539, Low）

`uv.lock` 再生成のみでは pytest 9.x が Python 3.10+ を要求するため、`requires-python = ">=3.9"` の下では Python 3.9 向けに pytest 8.x が解決され続け、将来も `< 9.0.3` 範囲のアラートが残る可能性がある。本対応で Python 3.9 サポートを終了する。

## 2. 方針

- 最小限の変更で Dependabot アラートを解消する。
- バージョン番号は SemVer に従い **0.1.0 → 0.2.0**（破壊的変更：Python 3.9 サポート終了）。
- 公開ドキュメントは整合性を優先し、Python バージョン記載を一括で更新する。
- OSS 公開観点で見つかった他の不整合（英語版文書の不足、CODE_OF_CONDUCT.md 不在、URL 誤り）は今回のスコープ外とし、別 Issue 化する。
