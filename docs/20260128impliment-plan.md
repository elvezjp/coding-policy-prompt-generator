## 4. リスクと対策

- リンク列が「説明」列と衝突し、補足が上書きされる問題
- 31文字制限や禁止文字でシート名が壊れる問題
- 既存の詳細シートが多数ある場合の命名衝突
- openpyxlが既存の数式・条件付き書式・グラフ等を完全に保全できない可能性
- READMEの手順が「まだ動かない状態」に見えにくい問題

対策（実装で担保すること）。

- `--link-column` を必ず指定可能にする
- `説明` と `リンク` が同一列のときは補足を扱わない方針を明文化する
- シート名の正規化と重複解決を入れる
- 既存シートを壊さず更新する方針を明文化する
- 出力は入力とは別ファイルに書き出す（既定: `<stem>_with_prompts.xlsx`）ことで、入力ファイルの破損リスクを回避する
- READMEに「現時点では未実装である」旨の注意書きを入れる
